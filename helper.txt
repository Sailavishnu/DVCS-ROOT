# Helper: Compile & Run JavaFX Client (Windows)

## 1. Prerequisites

1. Install JDK 17 or newer (from https://adoptium.net or Oracle).
2. Download JavaFX SDK for Windows from https://openjfx.io.
3. Unzip JavaFX SDK to a folder, e.g. C:\javafx-sdk-23.
4. Set environment variable (optional but recommended):
   - JAVAFX_HOME=C:\javafx-sdk-23

MongoDB auth commands are included in section 5.

---

## 2. Compile the JavaFX client (from dvcs-root)

### PowerShell (recommended)

Run this **in PowerShell** from `d:\dvcs-root`:

javac --module-path "$env:JAVAFX_HOME\lib" --add-modules javafx.controls,javafx.fxml -d client\out client\src\main\java\com\dvcs\client\MainApp.java client\src\main\java\com\dvcs\client\controller\LoginSignupController.java

### CMD (optional alternative)

If you prefer **Command Prompt (cmd.exe)** instead of PowerShell, you can use:

javac --module-path "%JAVAFX_HOME%\lib" --add-modules javafx.controls,javafx.fxml -d client\out ^
  client\src\main\java\com\dvcs\client\MainApp.java ^
  client\src\main\java\com\dvcs\client\controller\LoginSignupController.java

---

## 3. Run the JavaFX client

### PowerShell (recommended)

From `d:\dvcs-root`:

java --enable-native-access=javafx.graphics --module-path "$env:JAVAFX_HOME\lib" --add-modules javafx.controls,javafx.fxml -cp "client\out;client\src\main\resources" com.dvcs.client.MainApp

### CMD (optional alternative)

In Command Prompt (cmd.exe):

java --enable-native-access=javafx.graphics --module-path "%JAVAFX_HOME%\lib" --add-modules javafx.controls,javafx.fxml -cp "client\out;client\src\main\resources" com.dvcs.client.MainApp

---

## 4. Notes

- Always run the commands in sections 2–3 from the dvcs-root directory.
- If you see “Location is not set.”, ensure `client\src\main\resources` is on the runtime classpath.
- If you see `module not found: javafx.controls` or `javafx.fxml`, your JavaFX `lib` path is wrong (check section 5 `Test-Path` line).
- Prefer the section below to avoid relative-path and PowerShell line-continuation issues.

## 5. Copy/paste commands (PowerShell, robust)

### A) JavaFX (compile + run)

# Run these from ANY directory
$DVCS_ROOT = "D:\\dvcs-root"

# JavaFX SDK lib folder (recommended: set JAVAFX_HOME once)
$javafxLib = "$env:JAVAFX_HOME\lib"

# If `$env:JAVAFX_HOME` is NOT set, set $javafxLib to your full JavaFX `lib` folder, e.g.
# $javafxLib = "C:\\Users\\vishn\\Downloads\\openjfx-26_windows-x64_bin-sdk\\javafx-sdk-26\\lib"

# Quick sanity check (this MUST be True)
Test-Path "$javafxLib\javafx.controls.jar"

# Compile ALL .java (includes UI + auth)
javac --module-path "$javafxLib" --add-modules javafx.controls,javafx.fxml -cp "$DVCS_ROOT\client\lib\*" -d "$DVCS_ROOT\client\out" (Get-ChildItem "$DVCS_ROOT\client\src\main\java" -Recurse -Filter *.java).FullName

# Run JavaFX UI (includes FXML/CSS/resources on classpath)
# Optional: point the UI at your MongoDB (defaults to localhost + DVCS)
# $env:MONGODB_URI = "mongodb://10.210.141.117:27017"
# $env:MONGODB_DB = "DVCS"

java --enable-native-access=javafx.graphics --module-path "$javafxLib" --add-modules javafx.controls,javafx.fxml -cp "$DVCS_ROOT\client\out;$DVCS_ROOT\client\src\main\resources;$DVCS_ROOT\client\lib\*" com.dvcs.client.MainApp

### B) MongoDB auth demo (compile + run)

# Remote MongoDB (set env vars)
$env:MONGODB_URI = "mongodb://10.210.141.117:27017"
$env:MONGODB_DB = "DVCS"

# Compile only the auth package
javac -cp "$DVCS_ROOT\client\lib\*" -d "$DVCS_ROOT\client\out" (Get-ChildItem "$DVCS_ROOT\client\src\main\java\com\dvcs\client\auth" -Recurse -Filter *.java).FullName

# Run auth demo
java -cp "$DVCS_ROOT\client\out;$DVCS_ROOT\client\lib\*" com.dvcs.client.auth.AuthExampleMain
